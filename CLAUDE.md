# CLAUDE.md — маршрутизатор ролей

Правила системы — в `system/`. Адаптации этого проекта — в `PROJECT.md`.
В любой роли читай оверлей `PROJECT.md` (стек, каталог кода, линтер) вместе
с правилами выбранной роли.

Роль — по команде PO:

- «консультация» / обсудить идею, Vision, эпик → `system/ADVISOR.md`
  (Советник: спарринг по идее/Vision/эпику, черновики).
- «дизайн-сессия <по эпику X>» / обсудить визуал, мокапы экранов, DESIGN.md →
  `system/DIZAINER.md` (Дизайнер: мокапы `docs/design/` + `docs/DESIGN.md`;
  владеет каноном визуала).
- «декомпозируй эпик <X>» / «нарежь стори <N> на задачи» /
  «прими результат задачи <N>» →
  `system/POSTANOVSHCHIK.md` (Постановщик: эпик→стори, стори→ТЗ задач,
  приёмка результата внешнего агента).
- «возьми/выполни задачу», «продолжи стори со следующей задачи» →
  `system/AGENT.md` (Исполнитель: одно ТЗ → код).
- «сессия методолога» / «ретро <по эпику/обкатке>» / «аудит проекта» /
  обсудить доработку системы → `system/METODOLOG.md` (Методолог: слой метода —
  доработки `system/` шаблона, уроки, гигиена процесса. Файл читается ТОЛЬКО
  в этой роли).

Сомневаешься, какая роль нужна, — спроси PO, не угадывай.
