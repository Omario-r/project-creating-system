# Шаблон проекта: агентная разработка (PO + агент)

Универсальный каркас для разработки ПО с помощью ИИ-агента, где человек выступает
в роли Product Owner. Клонируй эту папку в начале каждого нового проекта.

## Структура

```
CLAUDE.md                     # Маршрутизатор ролей (Claude Code читает сам)
AGENT.md                      # Правила для агента-исполнителя — читается каждую сессию
ADVISOR.md                    # Правила агента-советника: идея, Vision, эпики
README.md                     # Этот файл (инструкция для PO)
GUIDE.md                      # Обзорный гайд: сценарии, антипаттерны (только для PO)
docs/
  VISION.md                   # Конституция проекта (пишет PO)
  ROADMAP.md                  # Очередь эпиков + уроки закрытых (советник и агент)
  STATUS.md                   # Дашборд: состояние проекта одним взглядом (ведёт агент)
  ARCHITECTURE.md             # Карта модулей (ведёт агент)
  DECISIONS.md                # Журнал ключевых решений (ведёт агент)
  BACKLOG.md                  # Лоток идей и багов (пишут оба, разбирает советник)
  templates/
    EPIC.md                   # Шаблон эпика
    STORY.md                  # Шаблон стори
    TASK.md                   # Шаблон задачи (встраивается в стори)
    MODULE.md                 # Шаблон паспорта модуля
  epics/
    <имя-эпика>/
      EPIC.md
      stories/
        01-<имя-стори>.md
src/
  <модуль>/
    README.md                 # Паспорт модуля
    ...код
```

## Быстрый старт нового проекта

1. Скопируй папку шаблона, переименуй под проект.
2. Заполни `docs/VISION.md` — сам или в сессии-консультации
   («Прочитай ADVISOR.md. Помоги мне сформулировать Vision проекта»).
3. Создай первый эпик — сам по шаблону или в сессии-консультации
   («Прочитай ADVISOR.md. Хочу обсудить новый эпик: ...»).
4. Дай агенту команду: «Прочитай AGENT.md. Декомпозируй эпик <имя> в стори».
5. Утверди список стори (**Гейт 1**).
6. Дай команду: «Возьми стори <N> в работу». Дальше агент сам.
7. Принимай стори по критериям (**Гейт 2**), отвечай на эскалации.
8. Все стори закрыты → ревью эпика (**Гейт 3**) → следующий эпик.

## Твой рабочий цикл (PO)

```
Vision → Эпик → [Гейт 1: утверждение стори] → агент работает
       → [ответы на «Открытые вопросы к PO»] → [Гейт 2: приёмка стори] × N
       → [Гейт 3: ревью эпика] → следующий эпик
```

Между гейтами система работает без тебя. Твои точки внимания:
- **Гейт 1** — 10 минут чтения экономят дни переделок. Проверяй логику продукта,
  не техничку: ничего не потеряно, ничего лишнего, порядок разумный.
- **Открытые вопросы** — отвечай быстро, они блокируют работу.
- **Гейт 2** — проверяй поведение по критериям приёмки, не код. Не принял —
  замечания агенту, он превратит их в задачи и вернётся на приёмку.
- **Идеи и баги** — не держи в голове и не вбрасывай в текущую работу:
  одна строка в `docs/BACKLOG.md`, разбор — на сессии советника.

## Типовые команды агенту

> В Claude Code префикс «Прочитай X.md» не нужен: CLAUDE.md маршрутизирует сам
> (по умолчанию исполнитель, слово «консультация» включает советника).
> Полные команды ниже — для любых других инструментов.

**Консультации (роль советника, ADVISOR.md):**
- `Прочитай ADVISOR.md. Сырая идея: <две фразы>. Помоги понять, стоит ли делать.`
- `Прочитай ADVISOR.md. Помоги мне сформулировать Vision проекта.`
- `Прочитай ADVISOR.md. Хочу обсудить новый эпик: <идея в двух словах>.`
- `Прочитай ADVISOR.md. Помоги решить, какой эпик брать следующим.`

**Исполнение (роль исполнителя, AGENT.md):**
- `Прочитай AGENT.md. Декомпозируй эпик <имя> в стори.`
- `Прочитай AGENT.md. Возьми стори docs/epics/<эпик>/stories/<файл> в работу: нарежь задачи и начни с первой.`
- `Прочитай AGENT.md. Продолжи стори <файл> со следующей задачи.`
- `Ответы на открытые вопросы: <...>. Обнови стори и продолжай.`

## Перенос существующего проекта (например, прототипа)

1. Скопируй код в `src/`.
2. Заполни `VISION.md`.
3. Команда агенту: «Прочитай AGENT.md. Проведи инвентаризацию src/: раздели код на
   модули, создай паспорт для каждого, заполни ARCHITECTURE.md. Кода не менять».
4. Проверь карту модулей — она станет фундаментом.
5. Дальше — обычный цикл с эпиков (первым эпиком часто стоит взять «приведение
   структуры к модульности», если прототип монолитный).
