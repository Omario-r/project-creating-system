# HANDOFF — правила сдачи для внешнего агента

> Дистиллят правил Исполнителя (`system/AGENT.md`) для агентов вне Claude Code
> (Gigacode, Cursor и др.): они правил системы сами не читают, а длинный свод
> слабая модель не удержит. Здесь — только несущее, нумерованным чек-листом.
> Полный контракт роли остаётся в `system/AGENT.md`; расходиться они не должны
> (правится шаблон, не проект).

## Промпт-запуск (канонический шаблон)

Постановщик, назначая внешний бэкенд, вписывает этот промпт **с подставленными
значениями** прямо в ТЗ задачи (поле «Промпт-запуск» — см. шаблон задачи);
PO копирует его оттуда в чат агента как есть, без правок.

```
Прочитай файл system/HANDOFF.md и следуй его чек-листу сдачи.
Выполни задачу T<N> — и только её — из файла docs/epics/<эпик>/stories/<стори>.md.
```

Если агент не умеет открывать файлы репозитория — вставь в его чат содержимое:
этот чек-лист + ТЗ задачи целиком.

## Чек-лист исполнителя (для агента)

1. Делай ровно то, что написано в ТЗ задачи («Что сделать»). За границы ТЗ не
   выходи: не рефактори соседний код, не добавляй фич, зависимости не добавляй.
2. Файлы, названные в ТЗ, читай из репозитория. Больше ничего не читай.
3. Есть поле «Арх-дельта» — примени его текст КАК ЕСТЬ (правки указанных в нём
   документов). От себя архитектурных решений не принимай.
4. Добавь тест, который проверяет «Результат» из ТЗ, — не только прогони
   существующие.
5. Добавил поле или параметр — проверь, что оно учтено везде, где объект
   сравнивается, копируется или перерисовывается (`equals`/`copyWith`/
   `shouldRepaint` и т.п.).
6. Верификация только детерминированная: команды линтера и тестов написаны в ТЗ;
   линтер — ноль ошибок и предупреждений, все тесты зелёные. Браузер, скриншоты,
   запуск приложения — не твоё (это проверят без тебя).
7. Обнови документы: паспорт каждого затронутого модуля (`README.md` модуля),
   файл стори (статус задачи + секция «Заметки по выполнению»), `docs/STATUS.md`.
8. Сделай ОДИН коммит со всем сразу: код + тесты + паспорта + файл стори +
   `docs/STATUS.md`. Ничего не оставляй незакоммиченным.
   Сообщение коммита: `<эпик>/<стори> T<N>: <суть>`.
9. Что-то в ТЗ неясно, противоречиво или невыполнимо — остановись и напиши об
   этом в ответе. Не додумывай.
