# ROSTER — каталог агентных ролей

> Генерик. Каталог ролей, между которыми Постановщик распределяет работу
> (поле «рекомендованный агент» в ТЗ). Здесь роли — как уровни-способности;
> конкретные модели/субагенты проекта — в `.claude/agents/` и `PROJECT.md`
> (см. «Привязка»).

## Роли

| Роль | Уровень | Что делает | Когда назначать |
|---|---|---|---|
| Советник | сильный | спарринг по идее/Vision/эпику | консультация |
| Постановщик | сильный | эпик→стори, стори→ТЗ, арх-доки, назначение | декомпозиция (прод-код не пишет) |
| Исполнитель | средний | одно ТЗ → код + тесты + локальные доки | по умолчанию для задачи |
| Исполнитель (тяжёлая) | сильный | то же, но задача сложна по существу | мудрёный алгоритм/рефакторинг даже при полном ТЗ |
| Критик | дешёвый | холодная вычитка ТЗ вслепую → список пробелов | сложные ТЗ до запуска Исполнителя |

## Флаг «тяжёлая»

Атрибут задачи, не отдельный архетип: тот же Исполнитель, но на сильной модели.
Ставит Постановщик, когда задача сложна ПО СУЩЕСТВУ даже при полном ТЗ (не путать
с «ТЗ недописано» — это эскалация, а не флаг).

## Холодная вычитка (Критик)

Для сложных ТЗ — флаг «тяжёлая», хрупкий домен (деньги/данные/расчёты/парсинг),
задача задевает архитектуру, крупная стори — дешёвый свежий агент читает ТЗ
вслепую и возвращает список пробелов ДО запуска Исполнителя. Свежие глаза видят
смысловые пробелы, которых не видит самопроверка автора. Оправдана там, где
реактивный возврат от Исполнителя дорог.

## Привязка к конкретным агентам (проект)

Уровни-способности → реальные модели: файл `.claude/agents/<роль>.md`
(frontmatter `model:`) + запись в `PROJECT.md`. Постановщик запускает
Исполнителя/Критика как субагентов Claude Code по полю «рекомендованный агент» ТЗ.
Пока конкретные агенты не заведены — роль называется словом, а исполнение идёт
обычной сессией по команде PO.
